Transparence & Méthode

Comment on calcule
le classement ?

Zéro magie noire. Voici exactement comment chaque score est calculé, de la collecte des données brutes jusqu'au rang final affiché.

Vue d'ensemble du système

🤖

Collecte automatique

Chaque lundi à minuit, un script Python tourne sur GitHub Actions et interroge les APIs de Spotify et YouTube.

🧮

Algorithme de scoring

Les données brutes sont normalisées puis combinées avec des coefficients pondérés pour produire un score final.

📡

Publication instantanée

Le fichier JSON résultat est commité dans le dépôt GitHub qui sert directement de CDN gratuit pour le site.

1

Quelles données on collecte ?

Quatre métriques réelles sont récupérées pour chaque artiste via des APIs officielles et gratuites :

🎵

Followers Spotify API officielle

Le nombre total d'abonnés de l'artiste sur Spotify. Indique la taille de la fanbase fidèle.

GET /v1/artists/{id} → followers.total

📊

Popularité Spotify Score interne 0–100

Score calculé en temps réel par Spotify basé sur le nombre d'écoutes récentes et leur fraîcheur. Reflète directement la viralité actuelle.

GET /v1/artists/{id} → popularity

▶️

Vues YouTube API Data v3

Cumul des vues des 3 derniers clips officiels de l'artiste. Mesure la consommation de masse au Maroc où YouTube est dominant.

search?q={artiste}+clip → videos?part=statistics → viewCount

🔥

Proxy Viralité Calculé

En l'absence d'API TikTok gratuite, on calcule un indice de viralité : popularité × √followers. Ce ratio isole les artistes qui explosent récemment sans avoir une très grande fanbase historique.

2

Normalisation MinMax

Le problème : Saad Lamjarred a 8 millions de followers Spotify, Khtek en a 400 000. Si on compare directement, Khtek serait toujours écrasé même s'il cartonne sur YouTube et TikTok. La solution : ramener toutes les métriques sur une échelle commune de 0 à 100.

// Formule de normalisation MinMax

score_normalisé = ( valeur_artiste − valeur_min )
                 ────────────────────────── × 100
                 ( valeur_max − valeur_min )

Exemple concret :

Max followers du marché : 8 000 000 (Saad Lamjarred)

Min followers du marché : 50 000 (artiste émergent)

Followers de Khtek : 400 000

Score Khtek = (400 000 − 50 000) / (8 000 000 − 50 000) × 100 = 4.4 / 100

→ Faible en followers, mais son score YouTube ou popularité peut compenser largement.

3

Les coefficients de pondération

Chaque métrique ne compte pas autant. On applique des poids qui reflètent l'importance réelle de chaque plateforme dans la consommation musicale marocaine :

▶️ Vues YouTube 35%

YouTube reste la principale plateforme de consommation musicale au Maroc. Les clips font des millions de vues localement.

🎵 Followers Spotify 25%

Mesure la fidélité long terme. Un artiste avec une grande base d'abonnés a une audience solide et récurrente.

🔥 Proxy Viralité (TikTok/Reels) 25%

Le ratio popularité × √followers capture les artistes qui explosent en ce moment. Un artiste "chaud" avec peu d'ancienneté peut scorer haut ici.

📊 Popularité Spotify 15%

Score temps réel de Spotify basé sur les écoutes récentes. Reflète l'activité actuelle, pas historique.

// Score final (0 à 100)

score_final =
  ( norm_youtube        × 0.35 ) +
  ( norm_followers     × 0.25 ) +
  ( norm_viralite     × 0.25 ) +
  ( norm_popularite   × 0.15 )

4

Calcul de l'évolution

Chaque semaine, l'algorithme compare le rang actuel avec celui de la semaine précédente pour afficher le mouvement :

▲ 3

L'artiste a gagné 3 places par rapport à la semaine dernière

▼ 2

L'artiste a perdu 2 places par rapport à la semaine dernière

=

L'artiste est stable, même position qu'il y a 7 jours

NEW

L'artiste entre dans le classement pour la première fois

Limites connues & honnêteté

⚠️

Pas d'API TikTok officielle gratuite

TikTok n'offre pas d'API publique gratuite. Le proxy de viralité (vues YouTube / fans Deezer) est une approximation, pas une mesure directe.

⚠️

Spotify : données de popularité inaccessibles depuis 2024

Spotify a restreint l'accès aux données de followers et de popularité via son API publique (Client Credentials) en 2024. Les liens Spotify sont affichés dans l'interface, mais les chiffres de scoring proviennent de Deezer, qui offre les mêmes données sans restriction.

⚠️

Anghami : pas d'API publique disponible

Anghami ne propose pas d'API officielle publique. Des liens directs vers les profils Anghami sont affichés pour chaque artiste, mais les données chiffrées ne peuvent pas être intégrées automatiquement.

⚠️

YouTube = 3 derniers clips seulement

On mesure les 3 vidéos les plus récentes, pas le catalogue complet. Un artiste avec un seul hit récent peut scorer plus haut qu'un artiste avec 50 clips historiques.

⚠️

Liste d'artistes non exhaustive

Seuls les artistes présents dans notre liste sont analysés. Si un artiste est absent, son influence n'est pas comptabilisée.

ℹ️

Code 100% open source

L'intégralité du code de collecte et de calcul est disponible sur github.com/zeplintor/morocco-billboard-100. Vous pouvez auditer chaque ligne.
